Historia aplikacji webowych

  • wczesne lata 90-te: statyczne strony HTML
  • od ok. roku 1993: pierwsze skrypty generujące dynamiczne dokumenty HTML
    • CGI
    • PHP (1995)
    • ASP (1996)
    • Java Servlets (1997)
    • JSP (1999)
  • wczesne lata 2000: nowoczesne technologie komponentowe: JSF, ASP.NET
  • ok 1995 roku: AJAX - Asynchronous JavaScript and XML
    • W odpowiedzi na żądanie użytkownika, strona HTML może zostać częściowo zmieniona bez przeładowania całego dokumentu.
    • Dane potrzebne do przebudowania strony mogły być pobrane z serwera w postaci dokumentu XML.

Metoda AJAX w znaczący sposób poprawiła odczucia użytkowników.

Aplikacje webowe stały się bardziej "responsywne" i zaczęły przypominać aplikacje desktopowe.

AJAX został zaadaptowany przez niektóre technologie Server-Side, np JSF.

A gdyby każda aplikacja webowa składała się z jednego dokumentu HTML*?

*Dokument ten byłby modyfikowany przez skrypty JavaScript w zależności od interakcji z użytkownikiem

Architektura aplikacji Rich Web Application

  • Każdy widok aplikacji posiada kontroler, który zarządza jego stanem.
  • Każdy kontroler odpowiada za:
    • Obsługę akcji (żądań) zainicjowanych przez użytkownika (np naciśnięcie przycisku).
    • Wczytanie danych do zaprezentowania na widoku.
  • Przejście między widokami realizowane jest przez managera stanów.
  • Manager stanów odpowiada za:
    • Przekierowanie do odpowiedniego widoku.
    • Przepisanie ścieżki URL w pasku adresu przeglądaki (tak aby możliwe było wysłanie linku lub dodanie linku do zakładek).
  • Kontroler komunikuje się z serwerem najczęściej wykorzystując w tym celu protokół HTTP/REST.
